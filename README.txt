数字图像处理课程 — 通用任务规格（AI 任务说明书）

本文件适用于所有 hw 子任务。每个 hw/subN/README.md 只描述该次实验的功能需求，
本文件定义交付物格式、报告规范、自动化流程、验收标准等通用约束。

======================================================================
一、通用代码要求
======================================================================
1. 使用 C++11/14，必须使用底层文件 API（fopen/fread/fwrite/fseek），
   不得使用 OpenCV、Qt 等高层图像库。
2. 支持读取无压缩 BMP（至少 24 位与 32 位输入）。
3. 支持写出 BMP（统一写出 24 位）。
4. 具备异常处理：文件不存在、格式非法、位深不支持、数据截断等。
5. 输出运行信息：处理耗时，灰度统计（最小值、最大值、均值、标准差）。

======================================================================
二、必须交付的输出
======================================================================
每个 hwN/subM 目录下应包含以下内容：
1. src/          — C++ 源码文件。
2. build/        — 可执行程序（如可编译得到）。
3. scripts/      — 自动化脚本（基准测试 + 报告图片生成）。
4. assets/test_bmps/      — 测试 BMP（自动生成）及处理结果。
5. assets/report_images/  — 报告插图（对比图、统计图、耗时图）。
6. docs/reports/report_main.md    — 实验报告正文。
7. docs/reports/benchmark_data.md — 自动化数据表。

注意：首次提交时不需要单独的修补文档（patch_notes），
所有内容直接写入 report_main.md。
只有在收到反馈需要修订时才生成 patch_notes。

======================================================================
三、报告内容最低要求
======================================================================
1. 包含：摘要、引言、实验内容、实现方法、结果分析、结论与体会。
2. 对相关原理进行清晰说明（如文件结构、算法推导等）。
3. 给出可复现实验步骤（编译命令、运行命令、参数说明）。
4. 给出量化结果（至少两组分辨率样例，含耗时和灰度统计）。
5. 插入自动生成的对比图与统计图，并给出简短图注。

======================================================================
四、报告格式约束（重要）
======================================================================
1. 报告正文只使用一级和二级标题。
2. 一级标题格式：`一、`  `二、`  `三、` …
3. 二级标题格式：`1.`  `2.`  `3.` …（论文风格短语）
4. 写法示例：
      四、实现方法
      1. 模块和接口设计
      <自然段正文>
5. 不使用三级及以上标题，不使用多级分点；更细内容写成自然段。
6. 报告不少于 2.5k 字，提交前使用 wc -m 统计字数并记录。

======================================================================
五、Word 版报告排版要求（来自课程实验要求原文）
======================================================================
报告全部用宋体。
报告题目：小三黑体居中。
正文：四号宋体，段落首行缩进 2 个汉字，行间距 1.5 倍，字间距正常。
大标题：居左，段前 1.5 行，段后 1 行，中文数字标示，四号黑体。
小标题：随正文首行缩进 2 个汉字，段前 1 行，段后 0.5 行，
        阿拉伯数字标示，小四黑体。
页眉：学号姓名（居左）、《数字图像处理》结课实验报告（居中）、
      年/月/日（居右）。
页脚：任课教师姓名（居左）、页号（居右）。
页眉页脚：五号宋体。

======================================================================
六、报告正文结构（已验证模板）
======================================================================
以下为经过实际提交验证的报告骨架，每次 hw 照此结构填写。
一级标题为固定项，二级标题中标注 [固定] 的每次必须出现，
标注 [按需] 的根据具体任务自行增减。

  《数字图像处理》结课实验报告
  题目：XXX 实验（C++11/14，底层 API）

  摘要
  <一段话概括：做了什么、怎么做的、结果如何>

  引言
  <背景意义、为什么选底层实现、本实验在课程中的位置>

  一、实验内容
    1. 实验目标与范围          [固定]  — 做什么、输入输出边界
    2. 方法对比与选择依据      [固定]  — 至少对比两种路线并说明选择理由

  二、实现方法
    1. 模块与接口设计          [固定]  — 整体架构、函数职责、命令行接口
    2. <核心算法/处理细节 A>   [按需]  — 根据任务展开，每个关键算法一节
    3. <核心算法/处理细节 B>   [按需]
    N. 可复现实验步骤          [固定]  — 编译命令、运行示例、脚本用法

  三、实验结果与分析
    1. 实验环境                [固定]  — 必须逐项列出以下硬件与软件参数：
         - CPU 型号与主频（如 Intel Core i5-3230M @ 2.60GHz）
         - 内存容量（如 8GB DDR3）
         - 显卡型号（如 AMD Radeon HD 7650M / Intel HD 4000）
         - 操作系统版本（如 Windows 10 22H2 / Ubuntu 25.10）
         - 编译器及版本（如 MSVC 14.44 / GCC 15.2.0）
         - 编译标准与优化等级（如 C++14, -O2）
         老师对此项要求严格，务必写全，不要省略
    2. 结果对比说明            [固定]  — 配合对比图，描述视觉效果
    3. 量化结果与性能分析      [固定]  — 引用 benchmark_data.md 中的数据
    N. 局限性与改进思考        [固定]  — 当前方案的不足和后续方向

  四、结论与体会
    1. 结论                    [固定]  — 客观总结实验完成情况
    2. 心得体会                [固定]  — 个人收获、对课程的认识

======================================================================
七、AI 自动化执行流程
======================================================================
1. 读取本文件（通用规格）和对应 hwN/subM/README.md（功能需求）。
2. 完成 C++ 代码并编译验证。
3. 生成测试 BMP 样例（小 256x256 / 中 1024x768 / 大 1920x1080）。
4. 批量运行程序，自动收集耗时与灰度统计，输出 benchmark_data.md。
5. 自动生成报告图片（对比图、耗时柱状图等），保存到 assets/report_images/。
6. 生成实验报告正文 report_main.md，回填自动化数据。
7. 使用 wc -m 统计报告字数，确认不少于 2k 字，但是也不应该多于 3k 字。
注意：不要删除已有文件，新增文件应清晰命名。
注意：图片相关操作一律在本地完成，不要传递到云端。

======================================================================
八、通用验收清单
======================================================================
1. 代码可编译可运行。
2. 24/32 位无压缩 BMP 输入处理正常。
3. 处理结果图可正常打开。
4. 自动化数据文件包含耗时与灰度统计。
5. 报告中有图、有数据、有方法、有结论。
6. 报告标题层级符合"仅一级和二级"要求。
7. 报告不少于 2.5k 字（wc -m 验证）。

======================================================================
九、编译器环境记录
======================================================================
1. Windows 环境可用 C/C++ 编译器：cl
   路径：C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Tools\MSVC\14.44.35207\bin\Hostx64\x64

2. Linux 环境可用 g++。
3. Python 环境可用；如果找不到请全局搜索。

======================================================================
1. 各 hw/sub 的功能任务是独立的，不要求功能点一致；
   但提交规格（代码、可执行、自动化数据、报告与图示）统一按本文件标准执行。
2. 图片处理限制：不要把图片传递到云端，所有图片操作在本地完成。
3. 首次提交时将所有内容直接写入 report_main.md，
   不需要单独创建修补文档；只有收到反馈后迭代时才生成 patch_notes。

BMP 图像读写实验（AI 任务说明书）

一、任务目标
1. 使用 C++11/14 实现 BMP 图像读写程序。
2. 必须使用底层文件 API（例如 fopen/fread/fwrite/fseek），不得使用 OpenCV、Qt 等高层图像库。
3. 根据 `dig实验要求.txt` 生成完整实验报告文字内容。
4. 自动生成实验数据与报告所需图片（对比图、统计图等），用于支撑结果分析。

二、输入文件
1. `dig实验要求.txt`：实验报告写作规范与提交要求。
2. 本 `readme.txt`：任务说明与验收标准。

三、必须交付的输出
1. C++ 源码文件（BMP 读写与处理）。
2. 可执行程序（如可编译得到）。
3. 实验报告正文文件（Markdown 或 txt）。
4. 修补/增补文档（用于根据反馈快速更新报告）。
5. 自动化数据文件（包含耗时、灰度统计等量化指标）。
6. 报告插图文件（原图/反色/灰度对比图，直方图，耗时图）。

四、代码功能最低要求
1. 支持读取无压缩 BMP（至少 24 位与 32 位输入）。
2. 支持写出 BMP（建议统一写出 24 位）。
3. 支持两种处理模式：反色、灰度化。
4. 具备异常处理：文件不存在、格式非法、位深不支持、数据截断等。
5. 输出运行信息：处理耗时，灰度统计（最小值、最大值、均值、标准差）。

五、报告内容最低要求
1. 包含摘要、引言、实验内容、实现方法、结果分析、结论与体会。
2. 对 BMP 文件头、信息头、像素区、行对齐、存储方向进行清晰说明。
3. 给出可复现实验步骤（编译命令、运行命令、参数说明）。
4. 给出量化结果（至少两组分辨率样例，含耗时和灰度统计）。
5. 插入自动生成的对比图与统计图，并给出简短图注。

六、格式约束（重要）
1. 报告正文只使用一级和二级标题。
2. 一级标题示例：`一、`；二级标题示例：`1.`。
3. 不使用三级及以上标题，不使用多级分点；更细内容写成自然段。

七、自动化建议流程（给 AI 执行）
1. 先完成 C++ 代码并编译验证。
2. 生成测试 BMP 样例（小/中/大三种分辨率）。
3. 批量运行程序（反色+灰度），自动收集耗时与灰度统计。
4. 生成结果表格文件。
5. 自动生成报告图片：
	原图/反色/灰度对比图、灰度直方图、耗时柱状图。
6. 生成实验报告正文并回填自动化数据。
7. 生成修补文档，便于后续根据反馈快速修改。

八、给下一个 AI 的直接指令模板
请先读取 `readme.txt` 与 `dig实验要求.txt`，然后一次性完成以下任务：
1. 产出符合 C++11/14 的 BMP 底层读写代码（含反色、灰度、异常处理、耗时与灰度统计输出）。
2. 编译并验证代码可运行；若环境缺失，说明缺失项并给出可执行命令。
3. 自动生成测试 BMP、自动跑数据、输出数据表。
4. 自动生成报告所需图片（对比图、直方图、耗时图）。
5. 生成实验报告正文（仅一级和二级标题）。
6. 生成修补文档，便于根据反馈继续迭代。
注意：不要删除已有文件，新增文件应清晰命名。

实验要求
《数字图像处理》实验报告
一、	实验报告内容
主要包含以下四部分内容：
1、	实验内容：根据所给题目简要叙述实验目的、常见的解决方法及各方法的优缺点等；
2、	实现方法：主要叙述研究方案、算法原理、程序设计思路、程序重难点解释等；
3、	结果分析：给出实验环境，如windows 10操作系统、主频、内存、DevC++/VScode等，实验结果，分析实验结果，比如算法效率、算法精度等，并给出相应的解释；
4、	结论与展望：根据实验结果分析得出相应的结论，比如方法的优缺点、适用场合等，并提出相应的改进策略或解决办法；心得体会，如对课程的意见和建议。

二、报告格式
报告全部用宋体，报告题目用小三黑体居中，正文四号宋体，段落右缩进2个汉字，行间距1.5倍，字间距正常；各内容大标题居左，段前1.5行，段后1行，用中文数字标示、四号黑体；小标题随正文右缩进2个汉字，段前1行，段后0.5行，用阿拉伯数字标示、小四黑体；页眉内容为：学号姓名（居左）、《数字图像处理》结课实验报告（居中）、年/月/日（报告时间，居右）；页脚内容为：任课教师姓名（居左）、页号（居右）；页眉页脚用五号宋体

## 二、原理与流程补强（补到“二、实现方法”）

### 1. 文件头与信息头关键字段细化说明
BMP 文件头（BITMAPFILEHEADER）中，`bfType` 用于标识文件类型，必须为 0x4D42；`bfOffBits` 指示像素数据起始偏移，是读取像素区前必须校验的关键字段。信息头（BITMAPINFOHEADER）中，`biWidth` 与 `biHeight` 表示图像尺寸，`biBitCount` 决定位深与每像素字节数，`biCompression` 指示压缩方式，本实验要求为 0（BI_RGB，无压缩），`biPlanes` 必须为 1。程序在读取阶段对上述字段进行逐项检查，确保后续像素解析逻辑成立。

### 2. 读写流程系统化描述
读入流程为：打开文件 -> 读取文件头/信息头 -> 校验关键字段 -> 根据 `bfOffBits` 定位像素区 -> 按位深计算每行步长 -> 逐行读取像素并转换为统一内存格式。写出流程为：构造输出文件头/信息头 -> 计算 24 位输出步长和填充字节 -> 按 BMP 底向上顺序写入像素行 -> 写入每行对齐填充。该流程的核心是“字段约束驱动像素读写”，任何关键字段异常都会立即触发错误并终止流程，避免错误扩散。

### 3. 24 位与 32 位输入差异补充
24 位 BMP 使用 BGR 三通道，每像素 3 字节；32 位 BMP 使用 BGRA 四通道，每像素 4 字节。程序读取时根据 `biBitCount` 动态分支，统一转换为内部 RGBA 结构，保证后续处理逻辑一致。输出阶段统一保存为 24 位 BMP，从而简化兼容性验证并减少不同查看器下的显示差异。

## 三、环境影响分析补强（补到“三、结果分析-实验环境”）

### 1. 环境因素对结果的可能影响
本实验运行环境为 Windows，C++11/14，VS Code，MSVC；CPU 主频 2.35GHz，内存 16GB。对本实验而言，环境因素主要影响运行时性能，不改变算法正确性：CPU 频率影响逐像素计算耗时，内存容量影响大分辨率图像处理时的缓存压力。由于算法主要为顺序内存访问和基础算术运算，不依赖并行加速，因此在不同硬件上结果图像内容应保持一致，但耗时数据会随硬件性能变化。

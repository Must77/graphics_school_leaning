《数字图像处理》结课实验报告

题目：图像增强实验（C++14，底层 API）

摘要

本实验基于 C++14 和底层文件 API 实现了四种灰度图像增强方法：对数增强、两种幂函数增强（gamma=0.4 与 gamma=2.5）以及直方图均衡化。程序支持 24/32 位无压缩 BMP 输入，输出 24 位 BMP。通过三组不同分辨率测试图进行基准测试，验证了各方法的正确性与效果差异。结果表明，对数和低 gamma 增强能提升暗区细节，高 gamma 增强压缩高亮动态范围，直方图均衡化的对比度拉伸效果最显著。

引言

图像增强是数字图像处理中的基本操作，目的在于改善视觉质量或使图像更适合后续分析。在医学影像、遥感和工业检测等领域，原始图像往往对比度不足，需要通过灰度变换来改善。本实验使用 C++ 底层文件 API 实现增强算法，深入理解 BMP 二进制结构和像素级操作原理。作为课程灰度变换与直方图处理部分的实践环节，通过三类方法的实现与对比加深对点运算理论的理解。

一、实验内容

1. 实验目标与范围

本实验要求实现四种图像增强子函数：对数增强、两种代表性幂函数增强和直方图均衡化。输入为 24 位或 32 位无压缩 BMP 图像，输出为灰度增强后的 24 位 BMP 图像。程序需具备命令行参数接口和交互模式两种使用方式，同时输出处理耗时和灰度统计信息（最小值、最大值、均值和标准差），以便量化评价增强效果。

2. 方法对比与选择依据

灰度增强的核心是建立输入到输出灰度的映射函数。本实验对比了三类方法：对数变换 s = c·log(1+r) 压缩高灰度、扩展低灰度，适合提升暗区细节；幂律变换 s = c·r^γ 通过 gamma 控制方向，小于 1 提亮暗区，大于 1 压缩暗区；直方图均衡化基于累积分布函数自适应扩展动态范围，无需调参。综合理论代表性和应用价值，选择以上四种方法进行对比。

二、实现方法

1. 模块与接口设计

程序整体由 BMP 读写模块、灰度增强模块和主控模块三部分组成。BMP 读写模块沿用课程前序实验的结构体定义和文件操作流程，支持 24/32 位输入并统一输出 24 位 BMP。灰度增强模块包含四个独立的增强函数，均接收 BmpImage 结构体并返回增强后的新图像。主控模块负责参数解析、计时和统计输出。命令行接口为 `bmp_enhance <input.bmp> <mode> <output.bmp>`，mode 取值 1 至 4 分别对应对数增强、幂函数提亮（gamma=0.4）、幂函数压暗（gamma=2.5）和直方图均衡化。

2. 灰度变换算法实现

对数增强的变换公式为 s = c·log(1+r)，其中归一化常数 c = 255/log(256)，保证输入 255 时输出恰好为 255。幂律变换的公式为 s = c·r^γ，归一化常数 c = 255/255^γ。两种 gamma 值分别选取 0.4 和 2.5，前者使曲线上凸从而提升暗区像素值，后者使曲线下凹从而压缩高亮区域。所有变换均先将彩色像素转换为灰度值（采用 ITU-R BT.601 加权公式 0.299R+0.587G+0.114B），再进行灰度映射，最后将结果写入 RGB 三通道。

3. 直方图均衡化实现

直方图均衡化的实现分为四步：首先统计输入图像的灰度直方图，然后计算累积分布函数 CDF，接着根据 CDF 构建灰度映射查找表 LUT[i] = round((CDF[i]-CDF_min)/(N-CDF_min)×255)，最后用 LUT 对每个像素进行映射。CDF_min 取第一个直方图非零项对应的累积值，N 为像素总数。该方法的计算复杂度为 O(N)，对任意大小图像都能高效执行。

4. 可复现实验步骤

编译命令：`g++ -std=c++14 -O2 -o build/bmp_enhance src/bmp_enhance.cpp`。运行示例：`./build/bmp_enhance input.bmp 1 output_log.bmp`（对数增强）。自动化基准测试：`python3 scripts/auto_bmp_benchmark.py`，该脚本自动生成三组测试 BMP（256×256 渐变图、1024×768 棋盘图、1920×1080 波纹图），对每组运行全部四种增强模式并记录数据到 docs/reports/benchmark_data.md。报告图片生成：`python3 scripts/generate_report_images.py`。

三、实验结果与分析

1. 实验环境

CPU：Intel Core i5-3230M @ 2.60GHz；内存：8GB DDR3；显卡：AMD Radeon HD 7650M / Intel HD 4000；操作系统：Ubuntu 25.10（Linux 6.17.0-19-generic）；编译器：GCC 15.2.0；编译标准与优化：C++14，-O2。

2. 结果对比说明

从对比图（见 assets/report_images/compare_*.bmp）可以直观观察到四种增强方法的差异。对数增强和 gamma=0.4 幂函数增强的效果较为接近，均显著提亮了暗区；但对数增强的提亮幅度更大，灰度均值从原始值显著上升。gamma=2.5 幂函数增强则明显压暗了图像，适合需要突出高亮细节的场景。直方图均衡化的效果最为特殊，它将灰度分布拉伸至近似均匀分布，使输出图像的均值接近 128 且标准差显著增大。从直方图对比图（histogram_before_after.bmp）可以清楚看到，均衡化前后灰度分布从集中变为分散。

3. 量化结果与性能分析

自动化测试数据见 benchmark_data.md。256×256 小图四种方法耗时均在 2-4ms 内。1920×1080 大图上，对数增强 78ms，幂函数 121-136ms（pow 计算量较大），直方图均衡化 53ms（查找表映射高效）。灰度统计方面，对数增强后均值约 219、标准差约 22，灰度压缩至高端；gamma=2.5 后均值降至约 58、标准差约 48，压缩至低端；直方图均衡化后均值 128、标准差约 73，实现最大灰度展宽。

4. 局限性与改进思考

当前实现将彩色图像先转灰度再增强，丢失了颜色信息，改进方向是在 HSV 或 YCbCr 空间仅对亮度分量增强。gamma 参数目前为固定值，可扩展为命令行可配置。直方图均衡化可能导致过增强，引入 CLAHE 可以缓解。

四、结论与体会

1. 结论

本实验成功实现了四种图像增强方法，所有输出图像均可正常打开。对数和低 gamma 幂函数适合提升暗图细节，高 gamma 适合压缩高亮区域，直方图均衡化适合自动化对比度增强。三组样例测试均通过，耗时与图像尺寸呈线性关系。

2. 心得体会

通过本实验，我对灰度变换的数学原理有了更深的理解。归一化常数 c 的计算是容易忽视但至关重要的细节。直方图均衡化让我体会到查找表方法的高效性。自动化基准测试流程使我能快速验证不同方法的效果差异。

《数字图像处理》结课实验报告

题目：图像均值滤波实验（C++14，底层 API）

摘要

本实验基于 C++14 和底层文件 API 实现了图像均值滤波，支持可配置的模板系数（全正值、正负混合、正负零混合）、模板大小、滤波次数、边界填充方式和移动间隔。程序支持 24/32 位无压缩 BMP 输入，输出 24 位 BMP。通过三组测试图在七种配置下进行基准测试，验证了各参数的影响。结果表明，增大模板或迭代次数可增强平滑但伴随细节损失，锐化和边缘检测模板展示了不同系数组合的应用。

引言

空域滤波是图像处理的核心操作，通过模板卷积对局部像素加权求和。均值滤波作为最基本的线性平滑滤波器，是理解卷积和模板设计的切入点。本实验使用 C++ 底层文件 API 实现可配置均值滤波，支持多种模板系数、可调大小、多次迭代、三种边界填充和可变步长，全面覆盖空域滤波的关键概念。

一、实验内容

1. 实验目标与范围

本实验要求实现图像均值滤波子函数，具体要求可配置以下五个维度的参数：模板系数（支持全正值的均值模板、含正负值的锐化模板、含正负零的边缘检测模板）、模板大小（3×3、5×5 等奇数尺寸）、滤波次数（单次或多次迭代）、边界填充方式（零填充、复制填充、镜像反射填充）以及模板移动间隔（步长）。输入为 24 位或 32 位无压缩 BMP 图像，输出为滤波后的 24 位 BMP 图像，同时输出处理耗时和灰度统计信息。

2. 方法对比与选择依据

空域滤波分为线性和非线性两类。本实验聚焦线性滤波框架，因为它通过不同模板系数可统一表达平滑、锐化和边缘检测。与频域滤波相比，空域滤波对小模板效率更高。边界处理实现了三种方案：零填充计算简单但引入暗边；复制填充避免暗边但边缘可能有偏差；镜像反射填充保持局部统计连续性，效果最佳。

二、实现方法

1. 模块与接口设计

程序由 BMP 读写模块、滤波核心模块和主控模块三部分组成。BMP 读写模块与前序实验保持一致。滤波核心模块实现了 getPixelPadded 函数用于带边界填充的像素访问，以及 meanFilter 函数执行多次迭代的模板卷积。主控模块解析七个命令行参数并协调整体流程。命令行接口为 `bmp_meanfilter <input.bmp> <output.bmp> <ksize> <kernel> <iterations> <pad_mode> <stride>`。kernel 参数支持逗号分隔的自定义系数或三个预置名称 avg、sharpen、edge。

2. 卷积与归一化实现

滤波核心采用标准的二维卷积计算：对以当前像素为中心的 ksize×ksize 邻域内每个像素，乘以对应模板系数后累加，最后除以系数总和进行归一化。归一化采用系数和而非固定值，因此对于全 1 的 3×3 均值模板等效于除以 9，对于锐化模板（中心为正、周围为负）则自动适配。当系数和接近零（如边缘检测模板）时，跳过除法直接截断到 0-255 范围。对于含负系数的模板，卷积结果可能出现负值，程序将其钳位到 0；同理超过 255 的值钳位到 255。R、G、B 三通道独立进行卷积计算。

3. 边界填充与步长实现

边界填充通过 getPixelPadded 函数统一处理：零填充返回 (0,0,0) 像素；复制填充将越界坐标钳位到 [0, width-1] 和 [0, height-1] 范围内；镜像反射填充将越界坐标关于边界进行对称映射。步长参数控制模板在图像上的移动间隔，当步长为 1 时输出分辨率与输入相同，步长为 2 时输出分辨率约为输入的一半。多次迭代时，每次迭代的输入为上一次迭代的输出，因此连续多次均值滤波相当于使用更大有效模板的高斯近似滤波。

4. 预置模板设计

程序内置三种预置模板：avg（均值模板，所有系数为 1，代表全正值情况）；sharpen（锐化模板，中心系数为 ksize×ksize，周围系数均为 -1，代表有正有负的情况）；edge（边缘检测模板，3×3 时为拉普拉斯算子 [0,-1,0;-1,4,-1;0,-1,0]，代表有正有负有零的情况）。用户也可通过逗号分隔字符串传入任意自定义系数。

5. 可复现实验步骤

编译：`g++ -std=c++14 -O2 -o build/bmp_meanfilter src/bmp_meanfilter.cpp`。均值滤波：`./build/bmp_meanfilter input.bmp out.bmp 3 avg 1 1 1`。锐化：将 avg 改为 sharpen。自定义模板：`./build/bmp_meanfilter input.bmp out.bmp 3 1,2,1,2,4,2,1,2,1 1 1 1`。自动化测试：`python3 scripts/auto_bmp_benchmark.py`。

三、实验结果与分析

1. 实验环境

CPU：Intel Core i5-3230M @ 2.60GHz；内存：8GB DDR3；显卡：AMD Radeon HD 7650M / Intel HD 4000；操作系统：Ubuntu 25.10（Linux 6.17.0-19-generic）；编译器：GCC 15.2.0；编译标准与优化：C++14，-O2。

2. 结果对比说明

从对比图（见 assets/report_images/compare_avg_filter.bmp）可以直观观察到均值滤波的平滑效果。原图中棋盘格的尖锐边界在经过 3×3 均值滤波 1 次后出现轻微模糊，3 次迭代后模糊程度明显增加，5×5 模板 1 次滤波的效果与 3×3 模板 3 次迭代相近。从锐化和边缘检测对比图（compare_sharpen_edge.bmp）可以看到，锐化模板增强了边缘对比度，而边缘检测模板则提取出了图像中的边缘轮廓信息，大面积均匀区域变为黑色。

3. 量化结果与性能分析

自动化数据见 benchmark_data.md。1920×1080 图像上，3×3 模板单次耗时约 145ms，5×5 约 309ms，与模板面积成正比。3 次迭代耗时约 379ms，步长 2 时降至约 51ms。灰度均值滤波后基本不变（约 127.5），标准差从 43.4 降至 39.1（3 次迭代）。边缘检测后均值降至约 10。replicate 与 reflect 填充仅在边缘像素处有微小差异。

4. 局限性与改进思考

当前直接卷积方式复杂度为 O(N·K²)，大模板效率较低，可用分离滤波器降至 O(N·K)。边缘检测模板系数和为零时未归一化，负值被截断有信息损失。改进方向包括：对边缘检测取绝对值或线性拉伸；实现高斯滤波作为加权改进版本。

四、结论与体会

1. 结论

本实验成功实现了参数可配置的均值滤波功能，涵盖三种模板系数类型，支持可变模板大小、多次迭代、三种边界填充和可调步长。21 组测试全部通过，定量验证了各参数对平滑程度和性能的影响规律。

2. 心得体会

通过本实验我对空域滤波卷积操作有了更深理解。边界处理是易错关键点，三种填充方式需仔细处理坐标映射。通过对比三类模板效果，我直观理解了系数正负性质如何决定滤波功能——正值平滑，负值微分，零值控制方向。
